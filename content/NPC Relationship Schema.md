# NPC Relationship Schema

This defines the canonical relationship block format for character pages.

## Relationship Types

- `ally`: cooperative and mutually reinforcing.
- `patron`: one-sided sponsorship or protection.
- `rival`: sustained competition with overlapping goals.
- `handler`: operational control without public authority.
- `dependent`: asymmetric reliance under pressure.
- `unknown-link`: suspicious association not yet validated.

## Relationships

- `type`: [[Target]] - one-line description of likely next shift.

## Rules

- Keep each line actionable, not abstract.
- Prefer direct links to named institutions and people.
- Track current pressure, not only historical ties.
